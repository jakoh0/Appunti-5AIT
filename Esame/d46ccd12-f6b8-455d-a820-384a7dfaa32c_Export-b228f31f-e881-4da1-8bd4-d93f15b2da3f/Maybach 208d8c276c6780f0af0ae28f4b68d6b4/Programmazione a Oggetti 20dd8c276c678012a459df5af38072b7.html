Programmazione a Oggetti

1. Concetti di base

  • Classe: blueprint o modello che definisce proprietà (variabili) e metodi (funzioni).
  • Oggetto: istanza di una classe, con il proprio stato.
  • Incapsulamento: racchiudere dati e comportamenti all’interno della stessa unità (classe), proteggendo l’accesso tramite visibilità.
  • Ereditarietà: una classe può estendere un’altra, ereditandone proprietà e metodi.
  • Polimorfismo: capacità di un metodo di comportarsi diversamente in base alla classe concreta (sovrascrittura/override, interfacce).

2. Definire una classe e creare oggetti

<?php
class Persona {
    // Proprietà
    public  $nome;
    public  $cognome;
    private $eta;

    // Costruttore
    public function __construct(string $n, string $c, int $e) {
        $this->nome    = $n;
        $this->cognome = $c;
        $this->eta     = $e;
    }

    // Metodo pubblico
    public function saluta(): string {
        return "Ciao, sono {$this->nome} {$this->cognome}.";
    }

    // Getter per l'età (incapsulamento)
    public function getEta(): int {
        return $this->eta;
    }

    // Setter per l'età con controllo (buona pratica)
    public function setEta(int $e): void {
        if ($e >= 0) {
            $this->eta = $e;
        }
    }
}

// Creazione di un oggetto
$p = new Persona('Mario', 'Rossi', 30);
echo $p->saluta();       // Ciao, sono Mario Rossi.
echo $p->getEta();       // 30
?>

4. Ereditarietà e Override

<?php
class Dipendente extends Persona {
    private float $stipendio;

    public function __construct($n, $c, $e, float $stp) {
        parent::__construct($n, $c, $e);
        $this->stipendio = $stp;
    }

    // Override del metodo saluta()
    public function saluta(): string {
        $base = parent::saluta();
        return $base . " Guadagno {$this->stipendio}€ al mese.";
    }
}

$d = new Dipendente('Luisa', 'Bianchi', 28, 2500.00);
echo $d->saluta();
// Output: Ciao, sono Luisa Bianchi. Guadagno 2500€ al mese.
?>

5. Interfacce e Polimorfismo

  • Interfaccia: dichiara un insieme di metodi senza implementazione.
  • Classe concreta: implementa l’interfaccia, garantendo la presenza di tutti i metodi.
<?php
interface LoggerInterface {
    public function log(string $msg): void;
}

class FileLogger implements LoggerInterface {
    public function log(string $msg): void {
        file_put_contents('app.log', $msg . PHP_EOL, FILE_APPEND);
    }
}

class DbLogger implements LoggerInterface {
    public function log(string $msg): void {
        // salva su DB
    }
}

function process(LoggerInterface $logger) {
    $logger->log("Process started");
}

process(new FileLogger());
process(new DbLogger());
?>

6. Trait (riuso di codice)

<?php
trait Timestampable {
    private DateTime $createdAt;
    private DateTime $updatedAt;

    public function setTimestamps(): void {
        $now = new DateTime();
        $this->createdAt = $now;
        $this->updatedAt = $now;
    }

    public function touch(): void {
        $this->updatedAt = new DateTime();
    }
}

class Articolo {
    use Timestampable;

    // ...
}

$a = new Articolo();
$a->setTimestamps();
sleep(1);
$a->touch();
?>

7. Metodi e proprietà statici

  • static: membro condiviso da tutte le istanze.
<?php
class Contatore {
    private static int $count = 0;

    public static function incrementa(): void {
        self::$count++;
    }

    public static function getCount(): int {
        return self::$count;
    }
}

Contatore::incrementa();
Contatore::incrementa();
echo Contatore::getCount();  // 2
?>

Esempio di __toString:

<?php
class Punto {
    private float $x, $y;
    public function __construct($x, $y) {
        $this->x = $x; $this->y = $y;
    }
    public function __toString(): string {
        return "({$this->x}, {$this->y})";
    }
}

$p = new Punto(3, 5);
echo $p;  // "(3, 5)"
?>

9. Best practices OOP in PHP

  1. SRP (Single Responsibility Principle): ogni classe fa una sola cosa.
  1. DRY (Don’t Repeat Yourself): evita duplicazioni tramite ereditarietà, interfacce, trait.
  1. Dependency Injection: passa dipendenze (es. logger) tramite costruttore o setter.
  1. PSR Standards: segui PSR-1, PSR-2/12 per lo stile del codice.
  1. Autoloading: usa Composer e PSR-4 per caricare automaticamente le classi.
